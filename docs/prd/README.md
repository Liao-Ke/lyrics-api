# 产品需求文档：歌词API

## 目标

基于 1647 首华语流行乐 LRC 歌词数据，构建一个**半开放只读 REST API**，供个人项目（音乐播放器、歌词插件、学习研究）查询歌词信息。

## 核心用户场景

1. **按条件查找歌曲**：根据标题、艺术家、作词人、作曲人过滤歌曲列表
2. **获取单首歌词**：拿到一首歌的完整歌词（含时间戳和翻译）
3. **全文搜索**：搜一句歌词正文找到对应的歌
4. **卡拉OK 时间轴**：根据当前播放时间点，获取当前行歌词

## 验收标准

| 场景 | 标准 |
|---|---|
| 搜索歌曲 | 输入标题/艺术家/作词人，返回匹配的歌曲列表，支持分页 |
| 获取歌词 | 返回完整歌词（time / text / translation），按时间戳排序 |
| FTS 搜索 | 输入歌词片段，FTS5 trigram 返回匹配歌曲 |
| 卡拉OK | 传入时间点，返回当前行 ± 前后各一行 |
| 鉴权 | 无 API key 返回 401 |
| 限流 | 超 RPM 返回 429 |
| 健康检查 | /healthz 返回 200 + db 状态 |
| 部署 | 裸跑和容器两种方式均可启动 |

## 不做范围

- 不提供音频文件或播放功能
- 不支持用户上传/编辑歌词（只读）
- 不做歌词社区（评论、点赞、分享）
- 不提供开放式公网无鉴权服务（半开放，需 API key）
- 不接入第三方音乐平台 API
- 不做多语言自动翻译（仅利用已有 translation 字段）
- 不做 OAuth / 用户系统（API key 管理即可）

歌词API

基于 1647 首华语流行乐 LRC 歌词的半开放查询 API，支持元数据搜索、FTS5 全文检索、卡拉OK 时间轴定位。

01 · 端点

API 端点

所有 /api/v1/* 端点需要 Authorization: Bearer <key> 头。健康检查 /healthz 无鉴权。

  1. GET /healthz健康检查

    返回服务状态和数据库连接状态，不限流不鉴权。Kubernetes/Podman 探针使用。

  2. GET /api/v1/songs歌曲列表

    分页返回歌曲列表，支持按标题、艺术家、作词作曲人模糊过滤。默认 20 条/页，最大 100 条。

  3. GET /api/v1/songs/{id}单首歌曲

    获取单首歌曲的元数据：标题、艺术家、作词人、作曲人、编曲人、是否有翻译。

  4. GET /api/v1/songs/{id}/lyrics歌词查询

    不传 ?time= 返回完整歌词（含时间戳和翻译）。传 ?time=<sec> 进入卡拉OK 模式，返回当前播放时间点前后各 1 行。

  5. GET /api/v1/search统一搜索

    FTS5 trigram 搜索歌词正文，SQL LIKE 搜索标题/艺术家/作词人。scope 参数控制搜索范围，逗号分隔。

02 · 示例

快速上手

# 获取 API key
podman exec -it lyrics-api python scripts/seed_key.py

# 搜索歌曲
curl -H "Authorization: Bearer <key>" \
  "http://localhost:8000/api/v1/songs?title=暗里着迷"

# 获取歌词（卡拉OK 模式，第 7.5 秒）
curl -H "Authorization: Bearer <key>" \
  "http://localhost:8000/api/v1/songs/1/lyrics?time=7.5"

# 全文搜索歌词
curl -H "Authorization: Bearer <key>" \
  "http://localhost:8000/api/v1/search?q=暗里着迷&scope=lyrics"

03 · 设计

架构概要

  1. 01
    FastAPI + pydantic-settings零 ORM，纯 sqlite3 标准库。依赖仅 4 个包：fastapi、uvicorn、pydantic-settings、loguru。
  2. 02
    FTS5 trigram 分词SQLite 内置分词器，零外部依赖。中文按三字滑窗，效果优于 unicode61 分词器。
  3. 03
    滑动窗口限流每请求独立判断，无固定窗口边界突刺。每 key 可配置独立 RPM，默认 60 次/分钟。
  4. 04
    多 API Key 鉴权SQLite 存储 key 哈希，支持签发/吊销。无需 OAuth 或用户系统，适合半开放作品集部署。
  5. 05
    内存 dict + TTL 缓存装饰器模式无侵入，1h 过期。支持换 Redis 只换装饰器实现，不修改 Repository 层。
  6. 06
    双模式部署裸跑 python -m app.main 快速迭代，Podman 容器作品集展示。多阶段构建，构建时生成 db 烤入镜像。

04 · 常见问题

FAQ

如何获取 API key？
容器部署：podman exec -it lyrics-api python scripts/seed_key.py。裸跑：python scripts/seed_key.py。输入名称即可获得 key。注意保存——key 哈希存储，不可找回。
搜索中文歌词支持多少字？
FTS5 trigram 对 3 字符及以上生效。1-2 字符的歌词搜索自动降级为 text LIKE。标题/艺术家/作词人搜索始终使用 LIKE，1 字符即可。
数据多久更新？
1647 首静态数据，按需更新。重新 build 镜像即可：podman compose build --no-cache && podman compose up -d。
容器部署需要什么环境？
Podman 5+ 或 Docker。单文件部署，开箱即用。镜像约 150MB，包含所有歌词数据。详情见 部署文档。
